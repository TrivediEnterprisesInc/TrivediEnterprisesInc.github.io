Desktop (Home and mobile app)

  • Integrated into the 365 apps launcher
  • All yr lists in one place (web/mobile)
  • Can Favorite a list to access it frm Favorites section

Views

  • Options incl List, Grid (list w/point-n-click editing), Gallery (CardVw) and Calendar.
  • Can Favorite a list to access it frm Favorites section
  • Views allow conditional formatting (eg. backcolor of item when status changes; font color + icons)
  • Security options: Personal views, Public vws (dumbed down)
ListView (DV)

GalleryView (CardDV)
  • Highlite lists containing images
  • Cards are configurable, disp a row of info

CalendarView
  • Pretty much the same as ours but notice the simplified UX, no skipping week/month/etc. Very few items on scrn.

Forms

Viewing an indiv list item in a form

Inside Teams

Can create Lists frm inside Teams

Rules

  • Bldg rules "as easy as writing a sentence".
  • Decide on outcome, click-fill if/then steps
  • Choose ppl, status, value changes to send notifications or update data.
  • Use rules to set reminders to keep ev.one informed.

Sharing

  • Can share a list w/Read-Only or Edit permissions
  • Can also share indiv items (DocLinks?) and allow/dis~ Edit, set an expiration date or require a pwd.

Templates

  • Provide templates for use cases incl.:
    • track issues
    • onboard new hires
    • event iternary
    • manage assets
  • Contain base struct, formatting, frms, conditional formatting
  • Find inspiration in them or modify em

Automation

  • Integrated w/Power Platform, so you can customize frms w/Power Apps or diz workflows w/Power Automate
  • The Lists API can be used as a data src via MS Graph

    • It is shiny from outside but has many issues that are embarrassing and can only be discovered once you have used it. (I guess looking at the open list of issues will also give you a hint).
      Their UI and interface isn't that bad but their APIs are VERY BASIC. For example, there is no way to retrieve metadata about an Airtable base (sheetnames, columns etc). Or, there is no way to tell whether a link between sheets is a singular link or list of links. No way to connect different bases together.
      Many of feature requests that were opened 3-4 years back still have no closure despite popular demand.
      In my opinion Airtable is alright for basic usage (like yours). But nothing advance.
      I know all this because I created an open-source library to dynamically create postgres database from multiple Airtable bases. (In case anyone is interested)
    • We run a fairly large business on it, and it's great from a UI standpoint, but for anything complex (such as reporting) we do dumps to Google Sheets where we run formulas to extract what we need. If I was starting all over, most likely I'd go directly for Google Sheets next time.
