Apr 23 ChatGPT conv: triggers/events Q:
I am building a custom database System. My client wants a custom (on-site) version of webHooks; which I am currently designing. I will design a "hook subApp" which takes (1) a condition/filter for records, selects recs according to (1); then runs an action (e.g., send an email, change a fieldValue in the same record). For instance, if a user wants to select all records with orderAmount > $100 and update the DiscountAmount field to give them a discount, the hook subApp will do it. What I need is more triggers/conditions/selection criteria. Since almost all webApps have hooks nowadays, could you pls list most commonly used EVENTS for such a use, as well as most common TRIGGERS (docUpdate/everyDay/etc.)? Thx

A:

Your custom database system and webhooks feature sound like a solid and useful tool! To help you build your trigger and condition options, here’s a list of common events and triggers that are widely used in various web applications for similar use cases.

Common EVENTS for Webhooks

These events typically relate to actions within the system, such as changes to data, user actions, or system-generated events:

  1. Data Creation/Insertion:

    • New record creation (e.g., new order, new user registration)
    • New comment, post, or review added
    • Record added to a specific table or category (e.g., new customer)

  2. Data Updates:

    • Update to a specific field (e.g., order status, inventory count)
    • Status change (e.g., from "Pending" to "Completed")
    • Update to related records (e.g., modifying a product, updating inventory after a sale)

  3. Data Deletion:

    • Record deletion (e.g., customer removal, order cancellation)
    • Soft deletion (e.g., "deleted" flag update, archival action)

  4. User Actions:

    • User login or logout
    • User registration
    • Password change request
    • User role or permissions change

  5. External Events:

    • API call or webhook trigger from an external service (e.g., payment gateway, social media post)
    • Integration with external tools (e.g., new item from an inventory management tool)

  6. Time-based Events:

    • Time-based conditions (e.g., every hour, every day)
    • Expiry date reached (e.g., subscription expired, discount expiration)
    • Periodic reminders or notifications (e.g., every X days, weekly)

  7. Business Logic Events:

    • Threshold exceeded (e.g., orderAmount > $100, or inventory quantity falls below a certain level)
    • Custom condition based on business logic (e.g., if the user has a VIP status, trigger a special offer)
    • Complex conditions like "If order amount > $200, and customer is from a certain region, trigger discount"

  8. System Errors/Alerts:

    • Errors or failed processes (e.g., failed transaction, failed API call)
    • System maintenance or downtime events
    • Overdue tasks or failed workflows

Common TRIGGERS for Webhooks

These are the actions that determine when the webhook should fire, usually based on certain conditions or system events:

  1. Data Changes:

    • After Insert: Trigger when a new record is created (e.g., after a new order is placed)
    • After Update: Trigger after an update occurs (e.g., after a user profile is modified)
    • After Delete: Trigger after a record is deleted (e.g., after a product is removed from the inventory)
    • Before Update: Trigger before a field value is updated (e.g., notify before changing the price of a product)

  2. Scheduled or Recurring Triggers:

    • Daily/Weekly/Monthly: Trigger based on a specific time interval (e.g., every day at midnight, every first Monday of the month)
    • Cron jobs: Advanced time-based triggers using cron syntax (e.g., every 5 minutes, at 3 AM daily)
    • Every X Minutes: Trigger at fixed intervals (e.g., every 10 minutes, every 1 hour)

  3. Threshold or Condition-Based Triggers:

    • Threshold: Trigger if a certain condition is met (e.g., when orderAmount > $500)
    • Field Value Comparison: Trigger when a field's value meets a specific condition (e.g., when status is "completed", when fieldValue is changed)
    • Cross-field Trigger: Trigger when two or more fields are compared against one another (e.g., if discountAmount exceeds orderAmount, then trigger)

  4. User Activity:

    • User Login: Trigger whenever a user logs in (e.g., send a welcome email or track user login)
    • User Logout: Trigger when a user logs out (e.g., notify admins of user activity)
    • User Registration: Trigger when a new user registers (e.g., send a welcome email or set up a user profile)
    • User Role Change: Trigger when a user’s role is modified (e.g., promote user to admin, or demote them to basic user)

  5. File/Document Changes:

    • File Upload/Download: Trigger when a file is uploaded/downloaded (e.g., send confirmation, track document access)
    • Document Approval: Trigger after a document or request is approved (e.g., notify the user, update order status)
    • File Modification: Trigger when a file or record is updated or modified (e.g., a contract update)

  6. System State Changes:

    • System Startup/Shutdown: Trigger when the system is started or stopped (e.g., notify admins of system status)
    • Maintenance Window: Trigger during scheduled maintenance periods (e.g., notify users about downtime)
    • Version Updates: Trigger when there is a new version of a system or API (e.g., alert for updates)

  7. Payment and Transaction Triggers:

    • Payment Success/Failure: Trigger when a payment is successful or fails (e.g., notify the user, update the order status)
    • Refund Processed: Trigger when a refund is processed (e.g., notify customer, update balance)
    • Subscription Renewal: Trigger when a subscription is renewed or canceled (e.g., update records, send reminder)

  8. Geolocation-Based Triggers:

    • Location-based triggers: Trigger when a user enters or exits a specific location (e.g., based on GPS, store check-ins)
    • Geo-fencing: Trigger when a user or device crosses a defined geographical boundary (e.g., check-in on app, arrival alert)

Example Scenarios for Triggering Webhooks

  1. Order Discounts: When an order amount exceeds a specific threshold (e.g., $100), update the discount field with a percentage.
  2. User Activity: Every time a user logs in, track the timestamp and update the last_login field.
  3. Inventory: Trigger when inventory quantity falls below a certain level, notifying the user to restock or reorder.
  4. Marketing Campaigns: When a customer makes their first purchase, trigger an email with a special promotion or discount.

Custom Conditions/Filters:

  • Date Comparison: Trigger when a record’s date field is within a specific range (e.g., sales that occurred this week, or past due invoices).
  • Field Range: Trigger when a field’s value is between two specific values (e.g., quantity between 50 and 100, order amount between $500 and $1000).
  • Multiple Conditions: Use AND/OR logical operators to combine different conditions (e.g., order amount > $100 AND customer country = “USA”).
  • Null/Empty Fields: Trigger when a required field is left empty or null.

By combining these events, triggers, and conditions, you can build a very flexible and dynamic webhook system that meets various needs, whether it's sending notifications, updating data, or integrating with other systems.

Certainly! You can use Node.js to evaluate JavaScript expressions from PowerShell by following these steps:
Install Node.js: from nodejs.org.
Create a JavaScript File: Create a JavaScript file (e.g., evaluate.js) that will evaluate the expressions. 
// evaluate.js
const vm = require('vm');
const expression = process.argv[2];
try {
    const result = vm.runInNewContext(expression);
    console.log(result);
} catch (error) {
    console.error('Error evaluating expression:', error.message);
}

Run the JavaScript File from PowerShell: Use PowerShell to run the Node.js script with the expression you want to evaluate. 
# Example expression
 $expression = "2 + 2"
# Run the Node.js script with the expression
node evaluate.js $expression
Example Usage
Save the above JavaScript code in a file named evaluate.js.
Open PowerShell and navigate to the directory containing evaluate.js.
Run the following command in PowerShell: 
node evaluate.js "5 * (3 + 2)"

This will output:
25
