Airtable

WebHooks: via external integrations / scripting or via automation sending HTTP requests. * UX: good formula editor with syntax highlighting, error checking. * Limitations: performance with deeply nested lookups / rollups; formula fields cannot directly mutate data (only compute); no “persistent scheduled recalculation every midnight” unless external trigger; limited ability to reference other bases except via sync. ---

Microsoft Lists / Power Apps / Power Fx

4. Specific “Today / volatile date” behavior & other tricky features

You mentioned one case: an Airtable user who couldn’t use Today’s date in a formula in a way it updates. That’s correct: many tools treat “today” or “now” as volatile functions that update only on refresh / record modification / view open etc. Let me compare similar behavior in other tools: * **Power Fx**: “Today()”, “Now()” etc will recalc when inputs change or the screen redraws etc. Usually quite reliable in canvas apps. * **Google Sheets**: “TODAY()” and “NOW()” are volatile; they often update on file open, on edit. They may not update exactly at midnight unless there is some trigger — users often have to force recalculation or have a script. * **Coda**: has functions for “Today” etc; behavior is reactive; dependency graph ensures updates under certain conditions. * **Baserow / NoCoDB**: support date functions; but volatile updates may have limitations (e.g. might not auto update unless record is edited or table is loaded/refreshed). Other tricky features: * Rollups / lookups across relations in formula columns: some tools allow it, some limit (esp. in Microsoft / Dataverse: formula columns may not yet support all types of referenced fields; rollups combined with formula columns sometimes constrained). * Reusability of formulas / “named formulas” etc is a good differentiator: MS Power Fx has it; Coda somewhat via named formulas / Packs; Airtable less so (can’t define reusable custom formula components beyond using same formula multiple times). * Debugging / editor support (error highlighting, intellisense, signature hints etc.) varies and becomes important as formulas grow. Power Fx editor has been improving; Baserow and NoCoDB less polished; Coda and Airtable decent. ---

5. Opportunities / gaps if building a competitive “No‑Code Database building app”

From this survey, here are features / behaviors that stand out as ways to differentiate or improve:
  • Truly volatile “today / now” / scheduled refresh without needing manual edits — tools that ensure date functions recalc automatically (daily or at configurable intervals) even if no record changed.
  • Named / reusable formulas or formula components — ability for users to define logic once and reuse in fields, buttons, filters etc.
  • Better editor UX — rich autocomplete, inline error hints; possibly natural‑language assist for formula writing; better performance for large formula sets.
  • Deep relational referencing — combining cross‑table references, rollups, lookups; formula columns that can operate across relations with strong performance.
  • Clear integration with Agents / WebHooks — ability for formulas to feed directly into automations / triggers / external calls in a low code way (for users who are not devs). Perhaps event (?) triggers that listen to formula field conditions etc.
  • Open / extensible function set — allow users (or plugins) to extend formulas; maybe allow custom functions or script functions but within sandbox; being OSS or close to open helps attract power users.
  • Performance / scalability — for many records (>10k / 100k), for many complex formulas / rollups, must remain responsive; UX of loading views with many computed fields needs care.

6. Final Thoughts & Ranking Justification

* I put **Microsoft Power Fx** first because it's the most mature in terms of combining expressive declarative formula logic, relation handling, integrations, and reuse. * **Coda** is close behind because of its flexibility and the fact formulas can live in many places, not only in table fields. * **Airtable** is strong for many use cases; older constraints (today volatility, cross‑base reference) are drawbacks. * **Baserow** and **NoCoDB** are appealing for open source / cost / control; formula features are good, though less polished. * **Google Sheets / Apps Script** is powerful, especially for general computational tasks, though sometimes the division between “formula field” (in a database table app) vs “spreadsheet” means different UX expectations. * **Retool** is more about building applications / dashboards; formula features tend to be more technical and less “formula field for every table” out of the box in a simple way. If your goal is to compete with MS Lists / Microsoft Power Fx, you’ll want to match or exceed in the features of reusability (named formulas), dynamic recalculation, relational references, and integration with agents + WebHooks, with excellent editor support. ---
