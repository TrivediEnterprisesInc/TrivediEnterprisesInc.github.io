3. Essential Features a Form Designer Should Have (Ranked by Importance)

  • Ease of Use: Simple drag-and-drop interface, intuitive controls, and guided workflows.
  • Field Variety: A wide range of field types such as text, number, dropdown, date, file upload, etc.
  • Custom Validation: Ability to create custom validation rules for fields, including conditional logic.
  • Conditional Logic: Show/hide fields or change values based on user input in real-time.
  • Design Customization: The ability to fully customize the look and feel of forms, including branding options.
  • Responsive Design: Forms should work seamlessly across all devices (desktop, mobile, tablet).
  • Integration with Other Apps: Integrate with CRMs, email marketing tools, Google Sheets, etc.
  • Collaboration Features: Allow multiple users to design and edit the form at the same time.
  • Multilingual Support: Ability to create forms in different languages for global reach.
  • Analytics and Reporting: Built-in features to analyze form submissions, generate reports, and track performance.

5. Drag-and-Drop Functionality Evaluation

Key Questions:

  • Does the app allow repositioning of fields via drag-and-drop?
  • Which app has the best UX for drag-and-drop?
  • What would the ideal drag-and-drop UX be for both novice and professional users?

Drag-and-Drop Features in Major Apps:

  • Coda: Coda allows users to drag and drop fields directly from a dialog panel to the form. It does allow repositioning of the fields, which is excellent for flexible form creation. However, some users report that the interface can be a bit crowded with multiple fields visible at once.
  • AirTable: AirTable’s drag-and-drop functionality is smooth. Fields can be easily moved around in the form layout but are limited to column-based structures, making it more suitable for users familiar with table-based views.
  • MS Lists: Users can create forms using PowerApps, but drag-and-drop is not as fluid compared to other apps. It does allow repositioning, but the workflow is not as intuitive as dedicated form apps.
  • JotForm: Known for its excellent drag-and-drop builder, JotForm allows full repositioning of fields within a form. The drag-and-drop experience is smooth, and users can easily reorganize elements in any part of the form. Additionally, the interface is well-designed with clear visual cues, making it suitable for both novice and pro users. It also offers an intuitive user flow, which reduces the learning curve for new users.
  • Typeform: Typeform’s drag-and-drop system is not as robust as others, primarily because it focuses on multi-step and conversational forms. However, it does allow users to drag and reorder questions within a section. The user interface is very clean and minimalist, and while it's user-friendly, it’s a bit restrictive when dealing with larger, complex forms.
  • Formly: Formly has a simple drag-and-drop interface that allows users to reposition form fields easily. The UI is clean, but the tool is more tailored for straightforward forms and might lack some advanced drag-and-drop flexibility seen in apps like JotForm or Coda.
  • BaseRow (OSS): BaseRow allows repositioning of fields using a drag-and-drop interface, but it can be a little cumbersome. The functionality is there, but some users report that the UI doesn’t feel as fluid or intuitive as those of more mature platforms like AirTable or JotForm.
  • NoCoDb (OSS): As an open-source tool, NoCoDb allows for drag-and-drop interactions, but the user experience is somewhat raw compared to proprietary tools. Customization options are vast, but they come with a steeper learning curve. Repositioning of fields is possible but not as seamless as more polished competitors.

Best UX for Drag-and-Drop:

JotForm leads the pack for the best UX in drag-and-drop form building. It strikes a balance between simplicity and functionality, offering users a smooth experience when moving fields around. The ability to reorder fields and manage large forms with ease makes it particularly favorable for both novice users and professionals.

Coda offers an excellent user interface, especially with its drag-and-drop support for tables, but its experience can feel cluttered when too many fields are visible at once. Coda's drag-and-drop functionality allows a high degree of customization, making it ideal for power users who need flexibility.

Typeform, while not as flexible, is still user-friendly and focuses more on conversational form creation. Its drag-and-drop feature is ideal for more basic forms but may not meet the needs of users looking to design complex layouts.

Ideal Drag-and-Drop UX for Novices and Pros:

The ideal drag-and-drop UX should cater to both novices and professional users:

  • For Novices: A clean, minimalist interface with clear visual cues, tooltips, and easily accessible features is key. Users should be able to drag fields into the form without much effort, with minimal distractions or complex customization options. A guided tutorial or onboarding flow is a plus to help users get up to speed.
  • For Professionals: Advanced users require more flexibility, including the ability to move fields, reorder sections, apply conditional logic, and create multi-step workflows. These features should be accessible without overwhelming the user interface, with options to customize the look and feel of forms further. Flexibility should not come at the expense of usability.

Conclusion

The No-Code form building space has evolved significantly over the years, with numerous competitors offering different strengths and weaknesses.
